Теоретические модули

🚀 Модуль 1: Архитектура Apache Spark

Apache Spark — это фреймворк для распределенных вычислений. Если данные не лезут на 1 сервер, мы делим их на части (партиции) и процессим на 10 серверах параллельно.

  • Driver vs Executor: Driver пишет план (DAG) и раздает задачи (Tasks), а Executors (Воркеры) их выполняют на конкретных партициях данных.
  • Lazy Evaluation: Трансформации (filter, withColumn) не выполняются сразу. Spark просто строит граф обработки (DAG). Реальное вычисление стартует только при вызове Action (show(), count(), write).

  • ▶️ YouTube: Лекции "Архитектура Apache Spark" (найти базовые объяснения Driver, Executor, Memory).
  • 📝 Документация: Cluster Mode Overview в официальной доке Spark.

📊 Модуль 2: DataFrame API и Трансформации

В основе PySpark лежат DataFrames, которые визуально напоминают Pandas, но принципиально отличаются ленивой природой и распределенностью.

Базовые операции: select, filter, groupBy, agg, join, withColumn (создание новой/замена колонки), drop.

Чтение/Запись: Spark умеет читать и писать множество форматов из коробки. Наиболее популярный и эффективный для Big Data — Parquet (колоночный формат), а также JSON/CSV во время прототипирования.

Что посмотреть/почитать:

  • 💻 Практика: Примеры из официальной доки Spark DataFrame API (PySpark).
  • ▶️ YouTube: Руководства "PySpark DataFrames Tutorial".
  • 📝 Статья: Отличия Parquet, ORC, Avro форматы (колоночное хранение).

🛠️ Модуль 3: Оптимизация (Партиции, Shuffle)

Боль, с которой сталкивается каждый ДЕ:

  • Shuffle (Перетасовка): Когда после groupBy или join данные должны физически перелететь по сети между экзекьюторами на другие ноды. Это самая дорогая операция!
  • Partitioning: Правильное разделение данных (repartition vs coalesce). Coalesce уменьшает число партиций без полного шаффла, repartition делает полный шаффл но равномерно балансирует.

Что посмотреть/почитать:

  • ▶️ YouTube: Spark Shuffle Optimization (как избегать shuffle и как работает BroadCast join).
  • 📝 Статья: "Coalesce vs Repartition в Apache Spark".
