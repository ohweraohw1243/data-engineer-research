Этап 1: Базовый SQL, Docker и Теория Хранилищ

Первые шаги: как обернуть базу данных в контейнер Docker, как писать SQL-запросы, и как устроена архитектура хранилищ (DWH).

Теоретические модули

🐳 Модуль 1: Docker для инженеров

Docker — это система, которая упаковывает ваш код (и базу данных) со всеми зависимостями в контейнер. Контейнер работает идентично везде: и на вашем Mac, и на корпоративном сервере Linux.

  • Image (Образ): Это "чертеж" или "рецепт". Как ISO-образ диска. Образы статичны и неизменяемы.
  • Container (Контейнер): Это запущенный экземпляр Образа. Он работает как отдельная изолированная мини-операционная система (свои папки, порты, сеть).
  • Dockerfile: Текстовый файл с инструкциями, как собрать свой личный Образ: "Возьми чистый Python 3.9, скопируй скрипт, установи requests".
  • Docker Compose: Инструмент для запуска нескольких контейнеров сразу в единой сети (docker network) с помощью файла docker-compose.yml.

🐘 Модуль 2: Реляционные БД и SQL (PostgreSQL)

Дата-инженер проводит половину жизни за написанием SQL. База данных — это программа, которая умеет эффективно хранить таблицы на диске и быстро отвечать на запросы.

Уровни владения SQL:

1. Базовый DML/DQL:

  • SELECT, WHERE, LIMIT: Выборка и фильтрация строк.
  • GROUP BY, HAVING: Агрегации (SUM, COUNT).
  • JOIN: Разница между INNER JOIN, LEFT JOIN и FULL OUTER JOIN.

2. Продвинутый (Инженерный) уровень:

  • Оконные функции: ROW_NUMBER(), RANK(), LAG(), LEAD(). В отличие от GROUP BY, окно считает агрегаты, сохраняя оригинальные строки.
  • CTE (WITH): Структурирование сложных запросов для читаемости.
  • DDL: Создание таблиц (CREATE TABLE), связей (Primary Key, Foreign Key) и типов (Integer, Varchar).

🏛️ Модуль 3: Архитектура DWH (Kimball & Stars)

Классическое хранилище данных (DWH) проектируется по методологии Ральфа Кимбалла.

Слои хранилища

  • STG (Staging Layer) — Сырой слой: Сюда мы просто переносим 1:1 данные из внешних источников (API). Данные не чистятся. Типы колонок часто тексты (Varchar), чтобы вставка не падала с ошибкой.
  • DDS (Dimensional Data Store) — Слой витрин: Ядро DWH. Данные очищены, типизированы и лежат по методологии "Звезда" (Star Schema).

Методология "Звезды" (Star Schema)

В центре "звезды" всегда лежит Таблица Фактов (Fact Table), а вокруг неё "лучами" присоединяются Таблицы-Измерения (Dimensions).

  • Факт: Действие (транзакция, клик). Содержит миллионы строк, состоит из внешних ключей (user_id, product_id) и метрик (сумма).
  • Измерение (Справочник): Описание. Отвечает на вопросы КТО, ГДЕ, КОГДА (Например: Пользователь, Товар, Дата). В ней содержатся текстовые атрибуты для фильтрации.
