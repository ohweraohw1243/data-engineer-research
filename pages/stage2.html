🐍 Тема 1: Конкурентность и Потоки в Python

Суть под капотом

Многопоточность (Threading) vs Многопроцессность (Multiprocessing): Из-за Global Interpreter Lock (GIL) потоки не ускоряют CPU-Bound задачи (вычисления, парсинг JSON), а нужны для I/O-Bound (парсинг сайтов, запись в БД). Для загрузки всех ядер используем Multiprocessing.

Асинхронность (asyncio): Однопоточная конкурентность (Event Loop), идеально для тысяч I/O соединений. Для БД используются пулы соединений как asyncpg (PostgreSQL).

Генераторы и память: Lazy evaluation (ленивое вычисление). Вместо загрузки DataFrame на 10 ГБ в ОЗУ, мы читаем файл по частям через yield (Chunking) для потоковой передачи в БД.

🛡️ Тема 2: Отказоустойчивость (Resilience) в Пайплайнах

Суть под капотом

Retry / Exponential Backoff: Сетевые ошибки (Timeout) могут починиться сами через пару секунд. Пишем декораторы, которые повторяют попытку через 2, 4, 8 секунд.

Идемпотентность: "При повторном вызове итог должен быть одним и тем же". Если код упал на середине, он не должен дублировать (INSERT) те строки, что уже вставил. Используйте "Upsert" (Merge) или "Delete/Truncate -> Insert".

ООП в пайплайнах: Инкапсуляция логики: классы Extractor, Transformer, Loader. Фабричные методы для подключения к БД (абстракции над Engine), логгеры и обработчики сбоев.

🐳 Тема 3: Инфраструктура, Docker & CI/CD

Суть под капотом

Контейнеризация (Docker): Упаковка скрипта и всех Python зависимостей (requirements.txt, Poetry) в изолированный контейнер (Image). Чтобы он запускался одинаково и у Васи на "винде", и на проде-Linux.

Kubernetes (k8s) основы: Оркестрация контейнеров. Понимание базовых абстракций: Pod (запущенный контейнер), Deployment (реплики подов для отказоустойчивости), CronJob (расписание для ETL).

CI/CD пайплайны: Как код попадает на сервера. GitLab CI или GitHub Actions. Линтеры (flake8, black, ruff) -> Unit тесты (pytest) -> Сборка Docker образа -> Выкатка в Registry -> Деплой.
