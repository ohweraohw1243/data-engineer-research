Этап 2: Python ETL и Сбор Данных

База данных сама себя не наполнит. В этом этапе учимся писать код на Python: делать API-запросы, парсить JSON, подключаться к PostgreSQL и запаковывать Python-скрипты в изолированные Docker контейнеры.

Теоретические модули

🌐 Модуль 1: Python, API и JSON

Чтобы стать Data Инженером, не нужно знать алгоритмы или писать сложные нейросети на Python. Достаточно понимать базовые структуры данных и уметь общаться с внешним миром по протоколу HTTP.

  • Библиотека Requests: Стандарт де-факто для HTTP-запросов. Вы делаете requests.get('https://api.example.com/data') и получаете ответ от чужого сервера.
  • JSON (JavaScript Object Notation): Главный формат обмена данными в интернете. Для Python формат JSON — это просто вложенные Словари (Dicts) и Списки (Lists).

💻 Практическая задача 1: Получение данных по API

Напишите скрипт на Python с использованием библиотеки requests, который обращается к открытому API CoinCap (https://api.coincap.io/v2/assets/bitcoin), получает JSON и выводит на экран текущую цену Биткоина (поле priceUsd).

💡 Показать подсказку

Не забудьте сделать import requests. Ответ от сервера нужно перевести в словарь методом .json(). Ключ с ценой лежит по скрытому пути: data["data"]["priceUsd"].

✅ Показать правильный ответ 
import requests

url = "https://api.coincap.io/v2/assets/bitcoin"
response = requests.get(url)
data = response.json()

price = data["data"]["priceUsd"]
print(f"Текущая цена Bitcoin: {price} $")

🔌 Модуль 2: Подключение Python к Базе Данных

У нас есть Python-скрипт с данными в оперативной памяти и запущенный PostgreSQL на порту 5432. Выступать связующим мостом будет "драйвер базы данных".

Низкоуровневый — psycopg2: Вы открываете соединение и пишете SQL-строки: cursor.execute("INSERT ..."). Важно: для загрузки массивов данных нужно использовать executemany().

Высокоуровневый — SQLAlchemy / Pandas: Просто кидаем DataFrame в БД через df.to_sql('table_name').

💻 Практическая задача 2: Массовый INSERT

Представьте, что у вас есть список кортежей: [ (1, 'Alice'), (2, 'Bob') ]. Используя сырой psycopg2, напишите код подключения к БД "my_dwh" (пользователь dev, пароль 123) на localhost, и вставьте эти данные в таблицу dim_users (id, username) за один запрос.

💡 Показать подсказку

Сначала вызовите cursor = conn.cursor(), а затем метод cursor.executemany("INSERT INTO dim_users VALUES (%s, %s)", data). Не забудьте conn.commit()!

✅ Показать правильный ответ 
import psycopg2

users_data = [(1, 'Alice'), (2, 'Bob')]

# Подключение к БД
conn = psycopg2.connect(
    dbname="my_dwh", 
    user="dev", 
    password="123", 
    host="127.0.0.1", 
    port="5432"
)
cursor = conn.cursor()

# Массовая вставка (executemany для множества строк)
insert_query = "INSERT INTO dim_users (id, username) VALUES (%s, %s);"
cursor.executemany(insert_query, users_data)

# Обязательно сохраняем изменения!
conn.commit()

cursor.close()
conn.close()

🐳 Модуль 3: Docker-изация Python скрипта

Мы не можем заставлять аналитиков вручную устанавливать Python, чтобы запустить ваш скрипт. Код нужно контейнеризировать!

Сети в Docker Compose (Важно!): Внутри Python-скрипта нельзя пытаться подключиться к базе по адресу localhost. Внутри контейнера с Питоном "localhost" — это он сам. Обращайтесь по имени сервиса из docker-compose.yml (например, host="postgres").

💻 Практическая задача 3: Изолируем скрипт

У вас есть папка со скриптом etl.py и файлом зависимостей requirements.txt. Напишите минимальный Dockerfile, который возьмет образ Python 3.10-slim, скопирует и установит библиотеки, а при старте контейнера выполнит ваш скрипт.

💡 Показать подсказку

Вам понадобятся директивы: FROM py..., WORKDIR /app, COPY ... ., RUN pip install ... и запуск скрипта через CMD ["python", "etl.py"].

✅ Показать правильный ответ 
# Используем минималистичный образ Python 3.10
FROM python:3.10-slim

# Создаем директорию /app внутри контейнера
WORKDIR /app

# Копируем список зависимостей
COPY requirements.txt .

# Устанавливаем библиотеки (requests, psycopg2 и т.д.)
RUN pip install --no-cache-dir -r requirements.txt

# Копируем сам скрипт обработки
COPY etl.py .

# Указываем команду при старте контейнера
CMD ["python", "etl.py"]
