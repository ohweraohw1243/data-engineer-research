Монорепозиторий от Нуля до PRO

Мы не делаем разрозненные примеры. Вы создадите один гигантский проект (Monorepo) на GitHub. С каждым этапом мы будем усложнять архитектуру, добавляя новые инструменты (Docker -> PostgreSQL -> Python -> Spark -> Airflow/Kafka). К концу у вас будет готовый End-to-End Data Pipeline.

🐘 Этап 1: Инфраструктура и Хранилище (PostgreSQL)

📂 Папка в репо: /stage1_database

Бизнес-задача: Заложить фундамент. Нам нужно поднять аналитическую базу данных с нуля на своем компьютере и спроектировать структуру будущих витрин нашего интернет-магазина (E-commerce).

План действий:

  • Инфраструктура: Установить Docker Desktop. Написать docker-compose.yml, который поднимает контейнер с PostgreSQL 15+ и контейнер с DBeaver (или pgAdmin).
  • Проектирование STG слоя: В DBeaver создать схему raw_data и таблицы (пока пустые): raw_users, raw_transactions. Типы данных — простые VARCHAR/TEXT.
  • Моделирование Кимбалла (DDS слой): Создать схему dwh. Написать DDL для таблицы фактов fact_sales и таблиц-измерений dim_users, dim_products. Связать их внешними ключами.
  • Фальсификация данных: Написать простые SQL-скрипты INSERT INTO со случайными данными (или COPY из CSV), чтобы протестировать джоины и оконные функции.

🐍 Этап 2: Python ETL и Заливка сырых данных

📂 Папка в репо: /stage2_python_etl

Бизнес-задача: Теперь у нас есть база. Нам нужен скрипт, который будет имититировать работу источника (тянуть данные по API) и выгружать их в сырой (STG) слой в PostgreSQL.

План действий:

  • Написать ETL скрипт с использованием библиотеки requests для запросов к открытому REST API.
  • С помощью psycopg2 или SQLAlchemy подключиться к локальному PostgreSQL в Docker.
  • Превратить полученный JSON в плоскую структуру и выполнить пакетную вставку (executemany) в таблицу raw_data.raw_transactions.
  • Завернуть скрипт в контейнер: Написать Dockerfile, чтобы скрипт запускался в отдельном Python-контейнере и общался с PostgreSQL через docker network.
Python API Requests SQLAlchemy JSON Dockerfile

⚡ Этап 3: Распределенные вычисления (PySpark)

📂 Папка в репо: /stage3_spark_processing

Бизнес-задача: Очистить сырые данные и загрузить их в витрины (DDS слой), используя движок для Big Data.

План действий:

  • Добавить в docker-compose.yml контейнер Jupyter с PySpark.
  • В Jupyter Notebook вычитать сырые таблицы из STG (PostgreSQL) в Spark DataFrame.
  • Очистить данные, отформатировать даты, сделать распределенные джоины со справочниками (с использованием Broadcast Hash Join для маленьких таблиц).
  • Записать готовый агрегированный DataFrame обратно в PostgreSQL (схема dwh).

🌊 Этап 4: Streaming, Kafka & Оркестрация (Airflow)

📂 Папка в репо: /stage4_streaming_airflow

Бизнес-задача: Перейти от ручного запуска скриптов к автоматизации пайплайнов по расписанию и добавить стриминг транзакций в реальном времени с контролем Data Quality.

План действий:

  • Брокер: Добавить кластер Kafka + Zookeeper (или KRaft) в docker-compose.yml. Инициализировать топики.
  • Producer & Consumer: Написать Python Producer для генерации потока покупок и Consumer, который пишет батчами в БД. Интегрировать проверки Great Expectations (DQ-тесты).
  • Оркестрация: Поднять Apache Airflow. Написать DAG, который каждый день в 00:00 запускает Batch PySpark-джобу (из Этапа 3) с ретраями при падении.

🏗️ Этап 5: Мультитенантный Data Lakehouse (S3 + BI)

📂 Папка в репо: /stage5_datalake_bi

Бизнес-задача: Системе перестало хватать PostgreSQL. Внедряем архитектуру Lakehouse. Настраиваем S3 (MinIO) как наше Data Lake и подключаем BI-дашборды для аналитиков.

План действий:

  • Озеро: Добавить локальный S3-совместимый MinIO. Настроить bucket'ы bronze, silver, gold.
  • Форматы: Переписать Spark джоб, чтобы он выгружал витрины в MinIO в формате Parquet или Apache Iceberg (партиционирование по датам/годам).
  • Деливерика: Развернуть BI-инструмент (Apache Superset или Metabase). Подключить его к нашим данным и собрать финальный интерактивный Дашборд продаж.
  • Финал: Запустить `docker-compose up`, который поднимет всё: от источника -> Kafka -> Airflow -> Spark -> S3 -> BI. Энд-ту-энд готов!
