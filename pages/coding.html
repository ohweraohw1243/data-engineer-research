📌 Кейс 1: Топ-3 по просмотрам в каждой категории

Задача: Найти топ-3 стримов в каждой категории.
Таблица: streams (id, name, category, views)
Подсказка 1: Используйте RANK() OVER (PARTITION BY category ORDER BY views DESC)
WITH ranked AS ( SELECT *, RANK() OVER (PARTITION BY category ORDER BY views DESC) as rank FROM streams ) SELECT * FROM ranked WHERE rank <= 3;

📌 Кейс 2: Скользящее среднее за 7 дней

Задача: Рассчитать 7-дневное скользящее среднее.
Таблица: daily_stats (date, views)
Подсказка: AVG() OVER (ORDER BY date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW)
SELECT date, views, AVG(views) OVER ( ORDER BY date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW ) as moving_avg_7d FROM daily_stats ORDER BY date;

📌 Кейс 3: Running total (кумулятивная сумма)

Задача: Рассчитать running total доходов.
Таблица: revenue (date, amount)
SELECT date, amount, SUM(amount) OVER (ORDER BY date) as running_total FROM revenue ORDER BY date;

🐘 Задача 2: JOIN + GROUP BY + HAVING

Есть таблицы orders(id, user_id, amount) и users(id, country). Напишите запрос, который считает по каждой стране: количество заказов и суммарный объем продаж. Оставьте только страны, где сумма продаж больше 5000. Отсортируйте по сумме продаж по убыванию.

SELECT
  u.country,
  COUNT(*) AS orders_cnt,
  SUM(o.amount) AS total_amount
FROM orders o
JOIN users u ON u.id = o.user_id
GROUP BY u.country
HAVING SUM(o.amount) > 5000
ORDER BY total_amount DESC;
✅ Показать эталон 
SELECT
  u.country,
  COUNT(*) AS orders_cnt,
  SUM(o.amount) AS total_amount
FROM orders o
JOIN users u ON u.id = o.user_id
GROUP BY u.country
HAVING SUM(o.amount) > 5000
ORDER BY total_amount DESC;

📊 Задача 3: Оконная функция (ROW_NUMBER)

Напишите запрос, который вернет последний заказ каждого пользователя из таблицы orders(id, user_id, amount, created_at) через оконную функцию ROW_NUMBER().

WITH ranked AS (
  SELECT
    o.*,
    ROW_NUMBER() OVER (
      PARTITION BY o.user_id
      ORDER BY o.created_at DESC
    ) AS rn
  FROM orders o
)
SELECT id, user_id, amount, created_at
FROM ranked
WHERE rn = 1;
✅ Показать эталон

💻 Проект 1: PySpark Streaming / Batch ETL

Цель: Написать скрипт на PySpark, который читает CSV файл (events.csv), оставляет только события 'purchase', группирует по 'user_id' и считает сумму покупок, затем сохраняет результат в Parquet партиционированно по 'date'.

events.csv:
user_id,event_type,amount,date
1,purchase,100,2026-05-01
2,click,0,2026-05-01
1,purchase,50,2026-05-01
✅ Показать эталон 
from pyspark.sql import SparkSession
from pyspark.sql.functions import col, sum

spark = SparkSession.builder.appName("ETL_Project").getOrCreate()

# Чтение
df = spark.read.csv("events.csv", header=True, inferSchema=True)

# Трансформации
result = df.filter(col("event_type") == "purchase") \
           .groupBy("user_id", "date") \
           .agg(sum("amount").alias("total_amount"))

# Запись (Parquet + partition)
result.write.partitionBy("date").mode("overwrite").parquet("output_data/")

spark.stop()

💻 Проект 1: Kafka Producer & Data Quality

Цель: Написать Kafka Producer на Python, который симулирует поток транзакций. Перед отправкой скрипт валидирует данные (DQ check): сумма (amount) должна быть > 0. Если данные невалидны — они отбрасываются.

События на генерацию:
{'txn_id': 1, 'amount': 150}
{'txn_id': 2, 'amount': -10} -- должно быть отброшено
{'txn_id': 3, 'amount': 200}
Топик: "transactions"
✅ Показать эталон 
import json
from kafka import KafkaProducer

producer = KafkaProducer(
    bootstrap_servers=['localhost:9092'],
    value_serializer=lambda v: json.dumps(v).encode('utf-8')
)

transactions = [
    {'txn_id': 1, 'amount': 150},
    {'txn_id': 2, 'amount': -10},
    {'txn_id': 3, 'amount': 200}
]

for txn in transactions:
    # DQ Check
    if txn['amount'] <= 0:
        print(f"DQ Alert: Invalid amount for txn_id {txn['txn_id']}")
        continue

    # Send to Kafka
    producer.send('transactions', value=txn)

producer.flush()
print("Valid transactions sent.")
